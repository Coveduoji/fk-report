# 安全运营周报生成

## 触发条件

用户说"生成周报"、"写周报"、"整理周报"时执行以下步骤。

## 步骤

### 1. 确定本周日期范围

今天日期可从系统获取。本周 = 本周一到本周五（或到今天，取较小值）。
日报文件命名格式为 `YYYYMMDD.md`（如 `20260701.md`）。

```bash
# 列出本周所有日报文件
ls ~/Desktop/weekly_reports/daily/
```

### 2. 读取本周日报

逐一读取本周范围内的日报文件（`~/Desktop/weekly_reports/daily/YYYYMMDD.md`）。
- 文件内容为口语化随手记，不强制结构
- 若某天文件不存在则跳过
- 若内嵌图片（markdown 图片语法），无法渲染时标注 `[附截图]`

### 3. 整合生成周报

将所有日报内容整合为专业格式，输出以下结构：

```markdown
# 安全运营周报

**周期：** YYYY年M月D日 — YYYY年M月D日

---

## 一、本周工作完成情况

| 类别 | 事项 | 备注 |
|------|------|------|
| [工作类型一] | 具体事项... | 进展/结果 |
| [工作类型二] | 具体事项... | 进展/结果 |

---

## 二、进行中事项

| 事项 | 状态 | 备注 |
|------|------|------|
| ... | ... | ... |

---

## 三、下周计划

| 序号 | 计划事项 |
|------|----------|
| 1 | ... |
| 2 | ... |

---

## 四、待解决问题 / 需要支持

| 问题 | 备注 |
|------|------|
| ...（若无则写一行"暂无"，备注列写"—"） | |

---

*生成时间：YYYY-MM-DD*
```

**整合原则：**
- 四个板块全部用表格呈现，不用标题+列表的形式
- "本周工作完成情况"按工作类型分类分行，不要按日期写流水账；同一类别下有多条事项时，
  同一类别可以合并成一行（事项列内部用分号或换行分隔），也可以拆成多行同类别
- 口语化表达润色为专业书面语，保留关键技术术语
- 跨天的同一任务合并为一条，在"备注"列标注进展
- 下周计划从日报中"明天/下周要做"的内容提取，按顺序编号
- 待解决问题从日报中卡点、等待、需要协助的内容提取，没有就保留表格只写一行"暂无"

### 4. 保存文件

将生成的周报保存到：
```
~/Desktop/weekly_reports/reports/周报-YYYYMMDD-YYYYMMDD.md
```
（文件名中的日期为本周一和本周五，格式 YYYYMMDD）

### 5. 告知用户

报告保存路径，简要说明本周工作亮点（1-2句话）。
